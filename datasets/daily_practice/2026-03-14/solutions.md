## Puzzle: Linear Multi-Touch Attribution

**The Scenario:** The marketing team wants to move away from the basic "Last-Click" attribution model. They want to implement a "7-Day Linear" model. This means that when a user makes a purchase, the revenue from that purchase should be divided equally among *all* the ad campaigns the user clicked on in the 7 days prior to the purchase.

**Your Task:** Write a query that calculates the total attributed revenue for each marketing campaign.

**The Rules:**

1. An ad click is "eligible" for a purchase if it occurred *before* the purchase, and strictly within the 7 days (168 hours) prior to the `purchase_time`.
2. The `purchase_amount` must be divided equally among all eligible clicks for that specific purchase.
3. If a purchase has *zero* eligible ad clicks prior to it, 100% of its revenue should be attributed to a default bucket named `'Organic'`.

The final report should show the `campaign_name` and the `total_attributed_revenue` (rounded to two decimal places), ordered alphabetically by campaign name.

**Expected Output:**

| **campaign_name** | **total_attributed_revenue** |
| ----------------------- | ---------------------------------- |
| Organic                 | 80.00                              |
| Retargeting_Banner      | 60.00                              |
| Search_Brand            | 50.00                              |
| Spring_Sale_Email       | 160.00                             |

### Tips for Approaching the Problem

1. **Join Purchases to Clicks:** Start with the `user_purchases` table and `LEFT JOIN` it to the `ad_clicks` table based on `user_id`.
2. **Apply the 7-Day Window:** In your join condition, ensure that the `click_time` is less than the `purchase_time`, AND that the `click_time` is greater than or equal to `purchase_time - INTERVAL 7 DAY`.
3. **Count the Touches:** For each purchase, you need to know how many eligible clicks it was matched with so you can divide the revenue. In a CTE, use the window function `COUNT(click_id) OVER (PARTITION BY purchase_id)` to get this denominator.
4. **Distribute the Value:** In that same CTE, calculate the fractional value. If the touch count is greater than 0, the value is `purchase_amount / touch_count`. If the touch count is 0, the value is just the `purchase_amount`.
5. **Handle Organic Traffic:** Use `COALESCE(campaign_name, 'Organic')` to rename the `NULL` campaigns generated by the `LEFT JOIN` for purchases that had no ad clicks.
6. **Aggregate:** Finally, group your results by the campaign name and `SUM` the distributed values.

**Your Solution:**

```sql
WITH combined_table AS (
	SELECT 
		ac.click_id, ac.click_time, ac.campaign_name, up.purchase_id, up.user_id, up.purchase_time, up.purchase_amount,
		COUNT(ac.click_id) OVER (PARTITION BY purchase_id) AS click_count
	FROM
		ad_clicks ac 
	RIGHT JOIN 
		user_purchases up ON up.user_id = ac.user_id 
		AND ac.click_time BETWEEN DATE_SUB(up.purchase_time, INTERVAL 7 DAY) AND up.purchase_time 
), 
revenues AS (
	SELECT 
		click_id, click_time, campaign_name, purchase_id, user_id, purchase_time, purchase_amount, click_count,
		CASE 
			WHEN click_count > 0 THEN purchase_amount / click_count ELSE purchase_amount 
		END AS revenue_share
		
	FROM
		combined_table
)
SELECT 
	COALESCE(campaign_name, 'Organic') AS campaign_name,
	ROUND(SUM(revenue_share), 2) AS total_attributed_revenue
FROM
	revenues
GROUP BY
	COALESCE(campaign_name, 'Organic') 
ORDER BY
    COALESCE(campaign_name, 'Organic');
```
